---
title: CSS options
description: Complete reference for all js-beautify CSS formatting options, including defaults and accepted values.
---

These options control how the CSS beautifier formats stylesheets. Pass them via the CLI (using `--kebab-case` flags), as an options object with `beautify.css()`, or inside a `.jsbeautifyrc` file under the `"css"` key.

## Indentation

<AccordionGroup>
  <Accordion title="indent_size" defaultOpen={false}>
    **CLI:** `--indent-size`, `-s` &nbsp; **Default:** `4`

    Number of spaces per indentation level. Ignored when `indent_with_tabs` is `true`.

    ```css
    /* indent_size: 2 */
    .selector {
      color: red;
      background: blue;
    }

    /* indent_size: 4 */
    .selector {
        color: red;
        background: blue;
    }
    ```
  </Accordion>

  <Accordion title="indent_char" defaultOpen={false}>
    **CLI:** `--indent-char`, `-c` &nbsp; **Default:** `" "` (space)

    Character used for each level of indentation. Use `indent_with_tabs: true` instead of setting this to `"\t"` directly.
  </Accordion>

  <Accordion title="indent_with_tabs" defaultOpen={false}>
    **CLI:** `--indent-with-tabs`, `-t` &nbsp; **Default:** `false`

    When `true`, uses a tab character for indentation and overrides `indent_size` and `indent_char`.

    ```css
    /* indent_with_tabs: true */
    .selector {
    \tcolor: red;
    }
    ```
  </Accordion>

  <Accordion title="indent_empty_lines" defaultOpen={false}>
    **CLI:** `--indent-empty-lines` &nbsp; **Default:** `false`

    When `true`, empty lines retain their indentation whitespace instead of being stripped.
  </Accordion>
</AccordionGroup>

## Line endings

<AccordionGroup>
  <Accordion title="eol" defaultOpen={false}>
    **CLI:** `--eol`, `-e` &nbsp; **Default:** `"\n"`

    Character sequence used for line endings. Common values: `"\n"` (LF), `"\r\n"` (CRLF).
  </Accordion>

  <Accordion title="end_with_newline" defaultOpen={false}>
    **CLI:** `--end-with-newline`, `-n` &nbsp; **Default:** `false`

    When `true`, ensures the output ends with a single newline character.
  </Accordion>
</AccordionGroup>

## Brace style

<Accordion title="brace_style" defaultOpen={false}>
  **CLI:** `--brace-style`, `-b` &nbsp; **Default:** `"collapse"`

  Controls where the opening brace of a rule is placed. The CSS beautifier supports two values:

  | Value | Description |
  |---|---|
  | `collapse` | Opening brace on the same line as the selector (default) |
  | `expand` | Opening brace on its own line |

  ```css
  /* brace_style: "collapse" (default) */
  .selector {
      color: red;
  }

  /* brace_style: "expand" */
  .selector
  {
      color: red;
  }
  ```
</Accordion>

## Selectors

<Accordion title="selector_separator_newline" defaultOpen={false}>
  **CLI:** `--selector-separator-newline`, `-L` &nbsp; **Default:** `true`

  When `true`, each selector in a comma-separated selector list is placed on its own line.

  ```css
  /* selector_separator_newline: true (default) */
  h1,
  h2,
  h3 {
      font-weight: bold;
  }

  /* selector_separator_newline: false */
  h1, h2, h3 {
      font-weight: bold;
  }
  ```
</Accordion>

## Rules

<Accordion title="newline_between_rules" defaultOpen={false}>
  **CLI:** `--newline-between-rules`, `-N` &nbsp; **Default:** `true`

  When `true`, an empty line is added between adjacent CSS rules.

  ```css
  /* newline_between_rules: true (default) */
  .foo {
      color: red;
  }

  .bar {
      color: blue;
  }

  /* newline_between_rules: false */
  .foo {
      color: red;
  }
  .bar {
      color: blue;
  }
  ```
</Accordion>

## Quick reference

| Option | CLI flag | Default | Type |
|---|---|---|---|
| `indent_size` | `--indent-size` | `4` | number |
| `indent_char` | `--indent-char` | `" "` | string |
| `indent_with_tabs` | `--indent-with-tabs` | `false` | boolean |
| `eol` | `--eol` | `"\n"` | string |
| `end_with_newline` | `--end-with-newline` | `false` | boolean |
| `brace_style` | `--brace-style` | `"collapse"` | string |
| `selector_separator_newline` | `--selector-separator-newline` | `true` | boolean |
| `newline_between_rules` | `--newline-between-rules` | `true` | boolean |
| `indent_empty_lines` | `--indent-empty-lines` | `false` | boolean |
